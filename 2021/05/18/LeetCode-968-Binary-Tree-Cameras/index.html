Posted on 2021-05-18 In LeetCode

LeetCode 968. Binary Tree Cameras

https://leetcode.com/problems/binary-tree-cameras/

說來慚愧，昨天為了趕OS的作業不小心斷更了QQ，今天努力一次生兩篇！

這題也是挺妙的，基本上可以先確定的是，與其在leaf節點上放camera，不如在leaf的parent節點上放，因為這樣能cover的節點比較多，很自然。
另外我們這裡是用preorder traversal來處理，這可能是這題最難想到的點吧…不過畢竟也只有三種方式遍歷，多試一下就出來了(?

之後分別用數字代表當前的節點狀態
no camera: 0
not need: 1
has camera: 2

if-else判斷條件:
1.如果我的左右兩個child有一個0以上，當前的root return 2(擺一個camera)來cover他的child
2.如果我的左右兩個child有一個2以上，當前的root return 1(被cover)
3.我的左右兩個child都是1(nullptr)，當前的root(leaf節點) return 0(不需要camera)
特別注意只有一個node時的edge case，當在main function call的help function回傳0時需要特別處理。
